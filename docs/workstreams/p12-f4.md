## Mission

Build F4, the federation SAR drafter. F4 receives an assembled package of
validated investigation artifacts and emits either a structured SAR draft or a
typed request for missing mandatory SAR inputs.

## Read First

- `AGENT_NOTES.md`
- `plan.md`, section P12

  `SARContributionRequest`, `SARDraft`
- `tests/test_messages.py`
- Existing agents: `backend/agents/a2_investigator.py`, `backend/agents/f3_sanctions.py`

## Inputs

- `SARAssemblyRequest`
- `SARContribution` records with optional `suspicious_amount_range` values in cents
- optional `GraphPatternResponse`
- optional `SanctionsCheckResponse`
- optional `PolicyEvaluationResult` records
- related query IDs

F4 should not fetch raw bank data and should not contact bank A2 agents
directly.

## Outputs

- `SARDraft`
- `SARContributionRequest` when mandatory SAR inputs are missing

## Expected Files

- `backend/agents/f4_sar_drafter.py`
- `backend/agents/prompts/f4_system.md`
- optional SAR helper/template module
- `tests/test_f4.py`
- UI/API readiness updates only if needed for F4 visibility

## Constraints

- Structured SAR fields should be deterministic where possible.
- Final `SARDraft.suspicious_amount_range` is computed from structured
  contribution ranges, not inferred from narrative text. Use the lowest
  provided low and highest provided high.
- Narrative can be LLM-authored but must stay within supplied facts.
- No raw customer names in narrative or request reasons.
- High priority is forced when sanctions or PEP evidence exists.
- Missing mandatory fields produce `SARContributionRequest`, not incomplete SAR output.

## Acceptance

- Complete S1-style input with contribution amount ranges emits `SARDraft` with
  mandatory fields.
- Sanctions or PEP evidence forces high priority.
- Missing mandatory data, including missing contribution amount ranges, emits
  `SARContributionRequest` addressed to F1.
- Narrative references Section 314(b), bank IDs, and hashes.
- Customer-name redaction tests pass.
